Refactoring Routines

The tool can be run in either of two modes; Workspace or File mode. If File mode has been selected from the toolbar, any refactoring functions selected will be applied to that single file only. On the other hand; if Workspace mode is selected, they will be applied to all source code files that matches the drop-down list of File extensions to parse, for all the selected folders in the grid and all their subfolders.

 There are two distinct groups of refactoring routines.

a) Refactoring Functions that make changes to the source code.

b) Report Functions that only report anomalies in the source code and writes out a list of affected source files.


Refactoring Functions


Remove Studio markers

Removes old "//AB" IDE markers. Earlier versions of the DataFlex Studio were called the 'IDE' and it used various place markers to separate  code areas. These are no longer needed and only clutter up the code base and it is therefor advised that they be removed.


Example:


//AB/ Object    // prj

//AB/ End_Object    // prj


//AB-StoreStart

Procedure Prompt_Callback integer hSL

    Set Initial_Column of hSL to 0

End_Procedure

//AB-StoreEnd



Remove Project object structure

Removes the 'Project Object Structure' comments at the top of source files. It also removes 'Register_Object' for those same objects. It is smart enough to only remove Register_Object lines for objects that are part of the Project Object Structure.


Example:


// Project Object Structure

//   BATCH is a dbView

//     Ac_DD is a DataDictionary

//     Art_DD is a DataDictionary

//     Artdisc_DD is a DataDictionary

//     Artgrp_DD is a DataDictionary

//     Cashhdr_DD is a DataDictionary

//     Cashline_DD is a DataDictionary

//     Cashlst_DD is a DataDictionary

//     Container1 is a dbContainer3d


Which later is followed by;


Register_Object BATCH

Register_Object Ac_DD

Register_Object Art_DD

Register_Object Artdisc_DD

Register_Object Artgrp_DD

Register_Object Cashhdr_DD

Register_Object Cashline_DD

Register_Object Cashlst_DD

Register_Object Container1


Remove Trailing spaces

Removes spaces at the end of lines


Remove End comments after End_Class/End_Object/End_Function/End_Procedure

Remove comments after End_Class/End_Object/End_Function/End_Procedure as they easily get out of sync, meaning that e.g. the object/procedure/function name is changed but adjusting the bottom end comment to match the new name is forgotten.


Example:


Procedure MyNewProcedure String sText

       Send Update

End_Procedure //MyProcedure


Remove 'Local' keyword from variable declarations

Removes the keyword: 'Local' from variable declaration lines in functions and procedures as they no longer apply.


Example:

Procedure Activate Returns Integer

   Local Integer iRetval iOk



Remove property keyword: Public

Removes the keyword: Public from property declarations as it is no longer serves any purpose.


Example:


Property String pSelStart1 Public ""

Property String pSelStop1  Public ""



Remove property keyword: Private*

Removes the keyword: Private from property declarations. Note that in some rare cases you might need to tweak your code if it is removed. Because of that reason, this option is unchecked by default.


Example:


Property String pSelStart1 Private ""

Property String pSelStop1  Private ""



Replace Calc/MoveInt/MoveReal/MoveNum/MoveStr with the Move command

In earlier DataFlex versions there existed several variants of the Move command, but they are obsolete they can be replaced with the correct Move command, making the code easier to read. The Studio's code complete syntax helper also understands the Move command, but none of the others.


Example:

Calc (iVar + 2) to iMyVar


Will be changed into into

Move (iVar + 2) to iMyVar


MoveStr "New Label" to sMyVar


Will be changed into into

Move "New Label" to sMyVar



Remove Unused local variables

Removes variables declared in functions and procedures that are never used.



Remove hard coded MS Sans Serif fonts

Removes hard coded: MS Sans Serif fonts if present. With more recent versions of DataFlex fonts are handled much better than before and these font settings usually makes the application look strange.



Change Get U_<Class Name> to (RefClass(<class>))

Replaces 'Get Create U_ClassName' with 'Get Create (RefClass(ClassName))'.


Example:

Get Create U_Array to hoArray


Is replaced with;

Get Create (RefClass(Array) to hoArray



Remove Blank lines

Blank lines can be good to make it easier to read the source code. To many blank lines thought makes it harder to read. How maximum blank lines there should be is a personal preference. The checkbox therefor have a spinform to the right where the Max blank lines can be set. The routine does not distinguish between objects and methods. The same max blank lines are enforced equally in the source.


Example:


//     order by name;

    

    Use Windows.pkg

    Use MSSQLDRV.pkg

    Use SQL.pkg

    

    

    Set Border_Style to Border_Thick

    Set Label to "Chapter 3 Sample (Microsoft SQL)"

    Set Location to 2 2

    Set Size to 172 299

    

    

    

    

    Object oResults is a cTextEdit


Will be changed into into

    //     order by name;

    

    Use Windows.pkg

    Use MSSQLDRV.pkg

    Use SQL.pkg

    

    Set Border_Style to Border_Thick

    Set Label to "Chapter 3 Sample (Microsoft SQL)"

    Set Location to 2 2

    Set Size to 172 299

    

    Object oResults is a cTextEdit



Rewrite single line If/Else constructs with Multiple lines

Restructures single if/else code lines to multiple lines, which makes debugging much easier. The first radio button will break the line with a space followed by a semi column. The second radio button will break the line with a semi column (no space in between). The third radio button will break the line by adding a begin / end block. The tab size governs how many spaces should be used when indenting the code block. Changing this setting changes how the editor (Editor view) indents. The same setting can be found on the Editor Properties dialog.


Example:

If (iRetval = 0) Procedure_Return


Which would be replaced to one of these three formats;

If (iRetval = 0) Begin

    Procedure_Return

End


Or;

If (iRetval = 0) ; // Space before the ";" character

    Procedure_Return


Or;

If (iRetval = 0);

    Procedure_Return



Editor Functions

Reindent* code

Re-indents the source file. You can change the Tab Size from the 'Editor Settings (Alt+D)' dialog.  NOTE: This is a Scintilla editor library function and will slow down the refactoring considerably.


Example:


    Use POTEXT.DD

    Use ARTSUB.DD


      DEFERRED_VIEW Activate_Art2Auto FOR ;

      Object Art2Auto is a DbView


        Procedure Activating

         Forward send Activating

         Set Value of oform2 to Comp.Mps_PoodAddr

         Set Value of oform3 to Comp.Mps_PopHeadtyp

        End_Procedure


Would with a tab-size of 4 be changed into:


Use POTEXT.DD

Use ARTSUB.DD


DEFERRED_VIEW Activate_Art2Auto FOR ;

    Object Art2Auto is a DbView


        Procedure Activating

            Forward send Activating

            Set Value of oform2 to Comp.Mps_PoodAddr

            Set Value of oform3 to Comp.Mps_PopHeadtyp

        End_Procedure




Adjust Upper/Lowercase*

Changes the case of DataFlex keywords for a source file. Note: You can change keyword spellings from the 'Editor Settings (Alt+D)' dialog to match your personal preferences. NOTE: This is a Scintilla editor library function and will slow down the refactoring considerably.


Example:

        If (iRec <> 0) Begin


Will be changed into into:

Drop Self*

Drop Self for the currently loaded file. In a view/dialog calling a method of an object no longer needs the (Self) keyword, so it can be removed making the code easier to read. NOTE: You will get a question to replace or not replace for each and every occasion of the 'Self' word. This is needed because at this moment the code that checks this does not know the difference between an object and a property.


Example of correct replacement:

Set Label of (oMyObject(Self)) to "New Label"


Will be changed into into

Set Label of oMyObject to "New Label"


However, this replacement would not work;

Set Enabled_State of (phoButton(Self)) to True


Will be changed into into

Set Enabled_State of phoButton to True


This particular change would break your code and that is something we do not want to do.

To alleviate the pain, the confirmation dialog that asks the question has a checkbox to remember the choice for the object you made a choice for, as you indicated it was an object or not.


Report Functions


Report Unused source files

Report source files that are never Used. Writes a list of all unused source files to a log file and shows them in a "Unused Source Files" grid dialog. It will NOT make any changes to the source, but from the dialog you can select which files (if any) should be moved to the backup area.
